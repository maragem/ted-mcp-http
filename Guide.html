What's in the server — all 7 tools

The server exposes 7 MCP tools. The two marked new were added in the latest version.

lookup_cpv_codes new
Search ~300 EU CPV codes by keyword. Always use this before searching to get the right 8-digit code.
search_notices
Structured search by keywords, country, CPV code and notice type. No raw query syntax needed.
get_notice
Retrieve full metadata for a specific notice by publication number.
get_latest_notices
Most recently published notices, filterable by country and CPV code.
download_notice
Returns HTML or XML content inline, or a direct PDF download URL.
get_notice_url
Returns the direct TED URL for a notice in any format and language.
read_notice_pdf
Downloads a PDF and extracts full text as clean Markdown using PyMuPDF4LLM.
summarise_notice new
Fetches metadata + PDF text in one call and instructs the LLM to produce a structured procurement analysis. No API key needed.
💡
No API keys required. All 7 tools work anonymously. The summarise_notice tool does not call an external AI — it returns the notice content to the LLM already in the conversation, which then generates the analysis directly.

Recommended workflow

The tools are designed to chain together naturally:

01
lookup_cpv_codes
Find the right CPV code for your sector — e.g. "drone" → 35613000 (Unmanned aerial vehicles)
02
search_notices / get_latest_notices
Search using the CPV code, country, and keywords to find relevant tenders
03
summarise_notice
Fetch full metadata + PDF text so the LLM can produce a complete procurement analysis in one step
04
read_notice_pdf / download_notice
Go deeper — read the raw PDF text or download the document in another language

Set up your local project

Your project folder should contain exactly these files:

ted-mcp-http/
  ├── server.py ← the MCP server (all 7 tools)
  ├── requirements.txt← Python dependencies
  ├── Dockerfile ← container build instructions
  ├── railway.json ← Railway config-as-code
  ├── TUTORIAL.html ← this file
  ├── README.md ← documentation
  └── .gitignore

Verify it runs locally first

terminal 
# Create virtual environment and install dependencies
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Start the server
python server.py

You should see:

INFO: Started server process [12345]
INFO: Application startup complete.
INFO: Uvicorn running on http://0.0.0.0:8000
Press CTRL+C to quit
⚠️
First install takes longer than usual. PyMuPDF includes native compiled binaries (~50 MB). This is normal — subsequent installs use pip's cache.
ℹ️
The MCP endpoint is at http://localhost:8000/mcp. Use the MCP Inspector to explore all 7 tools interactively before deploying.

Push to GitHub

Railway deploys directly from a GitHub repo. Every push to main triggers an automatic redeploy.

terminal 
cd ted-mcp-http
git init
git add .
git commit -m "TED MCP server — 7 tools"

git remote add origin https://github.com/YOUR_USERNAME/ted-mcp-http.git
git branch -M main
git push -u origin main
⚠️
Make sure .gitignore is in place. Never commit .venv/ or any .env files.
4

Deploy on Railway

  • 1
    Go to railway.app → sign in → click New Project.
  • 2
    Select "Deploy from GitHub repo". Authorise Railway if prompted.
  • 3
    Select ted-mcp-http from your repo list → click Deploy Now.
  • 4
    Railway detects the Dockerfile and starts building. The build takes 2–4 minutes because PyMuPDF includes native binaries — this is expected.
  • 5
    Once deployed, go to the service → Settings → Networking → "Generate Domain" to get your public URL.

Verify the deployment

terminal 
# Health check — should return HTTP 200
curl -I https://YOUR-DOMAIN.up.railway.app/mcp

# Explore all 7 tools in the MCP Inspector
npx @modelcontextprotocol/inspector https://YOUR-DOMAIN.up.railway.app/mcp

The MCP Inspector opens a browser UI where you can browse all 7 tools and run live test calls against the real TED API — no Claude connection needed.

Connect to Claude Desktop

Locate your config file

paths
# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json

Add the MCP server entry

claude_desktop_config.json 
{
  "mcpServers": {
    "ted": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://YOUR-DOMAIN.up.railway.app/mcp"
      ]
    }
  }
}

Restart Claude Desktop after saving. All 7 TED tools will appear in the tools panel.

Or connect from Claude.ai (web)

Go to Settings → Integrations → Add custom MCP server and paste: https://YOUR-DOMAIN.up.railway.app/mcp
